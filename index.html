It can't click for you

Copilot generates logic and steps. You still drag, drop, and build in Tableau. Like GPS: it navigates — you drive.

Unless you paste a screenshot or explain your fields, Copilot only knows what you tell it. More context = better answers.

It can't guarantee correctness

Always validate calculated fields against known numbers. Copilot gives a first draft — you make the final call.

Used well, Copilot makes a good Tableau analyst great. It doesn't replace the analyst.

K N O W  T H E  W A L L S  ·  T A B L E A U  L I M I T A T I O N S

Where Tableau stops. And what to do next.

Every experienced analyst has hit these walls. These are real, documented limitations — along with the exact Copilot prompts to navigate each one.

⚠ These are not bugs. These are by-design constraints you need to architect around.
LIMITATION 01 — Cannot Mix Aggregate & Non‑Aggregate

THE LIMITATION

You cannot combine SUM([Sales]) with row‑level [Sales] in the same expression.

WHY IT HAPPENS

Aggregate and row‑level calcs run at different granularities — Tableau cannot resolve both together.

RETAIL SCENARIO

You try SUM([Sales]) / [Sales] to get product share — Tableau errors out.

WORKAROUND

Use a FIXED LOD: {FIXED : SUM([Sales])} then divide aggregated values.

COPILOT PROMPT TO USE ▸ 
"I'm getting this Tableau error: 'Cannot mix aggregate and non-aggregate arguments.'
Explain why this happens and show me how to fix product-level share of sales using a FIXED LOD."
LIMITATION 02 — FIXED LOD Ignores Dimension Filters

THE LIMITATION

FIXED LODs compute before dimension filters — ignoring user selections.

WHY IT HAPPENS

FIXED LOD sits higher in Tableau's filter order — above dimension filters.

RETAIL SCENARIO

Region filter = "North", but FIXED company sales still show all regions.

WORKAROUND

Promote Region to Context or rewrite with INCLUDE.

COPILOT PROMPT TO USE ▸ 
"My FIXED LOD ignores Region filter. Explain why this happens and give two fixes:
(1) promote to Context, (2) rewrite using INCLUDE — with syntax."
LIMITATION 03 — Data Blending Only Supports Left Joins

THE LIMITATION

Blending always performs a left join from primary → secondary.

WHY IT HAPPENS

Blending is a post‑query merge at the granularity of the link field — not a database join.

RETAIL SCENARIO

Stores with no targets appear anyway (with NULL values).

WORKAROUND

Use Relationships/Joins or wrap secondary fields with ZN().

COPILOT PROMPT TO USE ▸ 
"I'm blending POS (primary) with Targets (secondary). Explain why blending only supports
left joins and how to fix NULL target values using ZN() or a Relationship instead."
LIMITATION 04 — Table Calcs Cannot Be Used Inside LODs

THE LIMITATION

You cannot nest RUNNING_SUM, RANK, WINDOW functions inside FIXED LODs.

WHY IT HAPPENS

LODs compute before the view; table calcs compute after the view is built — incompatible order.

RETAIL SCENARIO

You attempt RUNNING_SUM inside a FIXED LOD and Tableau errors out.

WORKAROUND

Compute LOD first → bring into view → apply table calc separately.

COPILOT PROMPT TO USE ▸ 
"Explain why table calcs cannot be nested inside LOD expressions and show how to restructure
the logic to compute cumulative sales correctly without nesting."
LIMITATION 05 — Parameters Are Not Dynamic Across Users

THE LIMITATION

Parameters have one global value and do not auto‑refresh with new data.

WHY IT HAPPENS

Parameters are static workbook variables unless configured as Dynamic Parameters.

RETAIL SCENARIO

New product categories do not appear until a developer republishes the workbook.

WORKAROUND

Use Dynamic Parameters (Tableau 2020.1+) or replace with filters.

COPILOT PROMPT TO USE ▸ 
"Explain why Tableau parameters don't auto-update with new data and walk me through
creating a Dynamic Parameter that refreshes from the data source."
LIMITATION 06 — COUNTD Is Slow and Sometimes Approximate

THE LIMITATION

COUNTD is one of the heaviest operations in Tableau.

WHY IT HAPPENS

Distinct counts require full de‑duplication — expensive on large datasets.

RETAIL SCENARIO

COUNTD(Customer ID) on 2M rows makes dashboard slow on live SQL.

WORKAROUND

Pre‑aggregate, use extracts, or leverage approximate count functions.

COPILOT PROMPT TO USE ▸ 
"My COUNTD(Customer ID) calc is slowing down my dashboard.
Explain why COUNTD is expensive and how to optimize using extracts or pre-aggregation."
LIMITATION 07 — Secondary LOD Requires Link Field in View

THE LIMITATION

Secondary-source LODs only work if the linking field is visible in the primary view.

WHY IT HAPPENS

Blending queries secondary data only at the level of the visible link field.

RETAIL SCENARIO

Your secondary FIXED LOD fails because Store ID (link field) is not on the view.

WORKAROUND

Add the linking field to Detail or restructure using joins/relationships.

COPILOT PROMPT TO USE ▸ 
"My FIXED LOD from a secondary source errors out unless the link field is on the view.
Explain why and show how to fix it without changing the chart visually."
LIMITATION 08 — Floating Point Measures Cause Comparison Errors

THE LIMITATION

Decimals like 0.32 may return as 0.320000001 — breaking equality tests.

WHY IT HAPPENS

Binary floating‑point values cannot represent many decimals exactly.

RETAIL SCENARIO

Two margins that should equal 0.32 compare as different in an IF statement.

WORKAROUND

Use ROUND() or tolerance ranges (ABS(x - y) < 0.0001).

COPILOT PROMPT TO USE ▸ 
"My LOD margin value is slightly off due to floating point precision.
Explain why and show how to fix comparisons using ROUND() or tolerance logic."
